Procesos / Transformaciones
Limpieza y validación
Nulos, duplicados, formatos de fecha y texto
Cálculo de campos derivados
— PERIODO (YYYYMM de FECHA ORIG) — AÑO_FEC_ORIG (año de colocación) — DIA ORIG (día del mes) — DIAS_UTILES_RESTANTES — FLAG CIERRE DE MES — MES2 / MES_FEC_ORIG
Clasificación de canales (RN-003)
DIG_DIRECTO Canal digital SCP_REFERIDOS Referidos SCP SCP_CLUSTER_DIRECTO Cluster ALIANZA Concesionarios
Clasificación de estados (RN-001 / RN-006)
TIPO: REAL / PRUEBA CERTIFICADO VALIDO?: SI / NO Estado Contrato: Activo / Retirado / Resuelto
Resolución de ejecutivos (RN-003)
USUARIO_EJECUTIVO → EJECUTIVO SCP (nombre real) vía BUSCARV sobre BD Lista Ejecutivos. Jerarquía: Ejecutivo → Jefatura → Supervisor.
Unificación BD_REFERIDOS2
Agenda Digital + Agenda Cluster → tabla unificada con campo AGENDA. Normalización STATUS (RN-009) y FRENTE_NORM (RN-008).
Acumulados y comparativos
Corte al mismo día del mes anterior para comparativos. Acumulado diario por día útil restante al cierre.

Filtros / Parámetros
Filtros Principales
• PERIODO (YYYYMM) • Estado Contrato   → Activo (certificados activos) • TIPO   → REAL (excluye pruebas) • FRENTE / CANAL   → DIG_DIRECTO   → SCP_REFERIDOS   → SCP_CLUSTER_DIRECTO • FECHA ORIG / DIA ORIG • AÑO_FEC_ORIG • CERTIFICADO VALIDO?   → SI (RN-006)
Parámetros / Lógicas
• Corte al mismo día del mes anterior para comparativos • FLAG CIERRE DE MES = "SI" para tabla días útiles • DIAS_UTILES_RESTANTES como agrupador • Redistribución leads SIN GESTIÓN al siguiente día hábil (referidos >6pm)
